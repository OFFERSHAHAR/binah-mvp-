ארכיטקטורת שכבות

9-Layer Architecture
L01 · Identity

שכבת זהות ומטרה

מגדיר את מטרת המוצר, תחום האחריות ורמת האוטונומיה. כל פעולה נשפטת לפי המטרה.

בודק: האם הפעולה תואמת את מטרת הסוכן?
L02 · Memory

Memory & Registry

זיכרון תפעולי: דפוסי כשל, תיקונים, גרסאות פרומפט, Thresholds וחוקים פעילים.

שומר כל שינוי עם timestamp ומשפיע על החלטות עתידיות
L03 · Input

Input Audit

בדיקת שלמות, פורמט ועקביות קלט. קלט כושל → עצירה + סימון שדות בעייתיים.

דורש TRUE ב-5 תנאים לפני המשך
L04 · Security

Injection Scan

מזהה Prompt Injection, פקודות מוסתרות, טקסט לבן ומסמכים עוינים.

חסימה מלאה + Builder Mode + לוג אבטחה
L05 · Truth

Truth Match

משווה כל פלט מול מקור אמת חיצוני. חוסר התאמה → חסימה ופתיחת חריגה.

Retrieval Speed: 0.1ms · Source: Google Sheets / DB
L06 · Output

Output Audit

ביקורת פלט לפני כל שליחה. Confidence נמוך → עצירה ובקשת אישור אנושי.

Threshold: 85% Confidence לאישור אוטומטי
L07 · Execution

Execution Flow

זרימת פעולה מבוקרת: Trigger ← Audit ← Scan ← Match ← Output.

כל שכבה חייבת לעבור לפני המעבר הבאה
L08 · Autonomy

Autonomy Levels

L0→L3 scale. פעיל ב-L1/L2. L3 דורש הרחבת הגנות.

השינוי ל-L3 דורש אישור מפתח + 30 יום Early Life
L09 · Creator

Creator Channel

דיווח למפתח, Kill Switch, Builder Mode ודוחות שבועיים אוטומטיים.

דוח שבועי: success, failures, blocked, self_score

Input & Output Audit

שכבות ביקורת

Input Audit — ביקורת קלט

כל קלט עובר 5 בדיקות לפני שמותר להמשיך. כשל אחד = עצירה מלאה.

✓שם מלא קיים ותקין
✓מספר טלפון בפורמט IL/INT תקין
✓מספר חדר תואם לרשומת הזמנה
✓תאריך הגעה / עזיבה לא סותרים
✓מזהה הזמנה קיים ופעיל
כשל → עצירה מיידית
לא נשלחת הודעה, נפתחת חריגה, מסומנים השדות הבעייתיים

Output Audit — ביקורת פלט

לפני כל שליחה או שינוי, הפלט עובר 4 בדיקות. Confidence נמוך = בקשת אישור.

✓הפלט תואם לתוצאות Input Audit
✓Confidence Score ≥ 85%
✓לא נמצאת סתירה מול Truth Match
✓הפעולה מאושרת לפי Autonomy Level
כשל → אישור אנושי
הפעולה מוקפאת עד שמנהל מאשר ידנית
